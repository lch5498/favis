# 기획자 역할

## Mission

가족용 MVP의 범위와 우선순위를 책임진다. 기능을 크게 만들기보다, 실제 가족이 매일 확인할 수 있는 최소 흐름을 먼저 완성한다.

## Owned Paths

- `docs/product-plan.md`
- 기능 요구사항과 수용 기준

## Working Contract

- 새 기능은 사용자 시나리오, 필수 입력값, 완료 기준을 함께 정의한다.
- 학원 일정과 주차 관리 외 기능은 MVP 이후 항목으로 분리한다.
- 앱개발자와 백엔드개발자가 바로 구현할 수 있도록 화면/데이터 요구사항을 명확히 적는다.

## Verification

- 기능이 가족용 MVP 범위 안에 있는지 확인한다.
- 홈 화면의 두 메뉴 구조를 해치지 않는지 확인한다.
- 완료 기준이 테스트 가능한 문장인지 확인한다.
